### **Аудит резюме: Аналитик данных**

**1) Сильные стороны**

*   **Опыт работы:** Солидный общий профессиональный опыт (9+ лет), что говорит о зрелости и опыте работы в бизнес-среде, даже если не весь он напрямую в аналитике.
*   **Актуальный стек навыков:** Владение ключевыми инструментами современного аналитика данных: **Python** (pandas, NumPy), **SQL**, **ClickHouse**. Это основа для работы с данными.
*   **Бизнес-ориентированность:** Наличие знаний в области ключевых бизнес-метрик (LTV, ARPU, Retention, ARPPU), маркетинговой и финансовой аналитики. Это критически важно для перевода данных в бизнес-ценность.
*   **Наличие портфеля проектов:** Ссылка на портфолио на GitHub — это мощное конкурентное преимущество, позволяющее наглядно продемонстрировать навыки кодирования, аналитический подход и умение решать задачи.
*   **Проактивность в обучении:** Завершены современные курсы по аналитике и автоматизации (ATI, Zerokoder) в 2025 году, что показывает мотивацию и стремление развиваться в профессии.

**2) Риски и слабые места**

*   **Полное отсутствие описания опыта работы:** Это главная проблема резюме. Рекрутер не видит, **где, когда и в какой роли** кандидат применял перечисленные навыки. Нет конкретных проектов, достижений, метрик влияния. Опыт выглядит "виртуальным".
*   **Неструктурированное и дублирующееся образование:** Блок "education" представляет собой нечитаемый текст с повторами. Создается впечатление небрежности. Указание нескольких "средних специальных" образований в 2025 году требует пояснения (это курсы, а не дипломы колледжа).
*   **Размытый заголовок и цель:** Название резюме и должность совпадают (`"Аналитик данных"`), но нет краткого "слогана" или цели (Summary), который бы сразу расставлял акценты (например, "Аналитик данных с фокусом на продуктовые и маркетинговые метрики").
*   **Проблемы с форматированием:** Поле `"salary"` содержит текст вместе с цифрой (`"Аналитик данных150 000 ₽ на руки"`). Поле `"about"` пустое.
*   **Список навыков требует чистки:** Навыки перечислены без структуры (Hard Skills, Soft Skills). Есть смысловые повторы ("Аналитика", "Анализ данных", "Data Analysis"), избыточные общие навыки ("Внимательность"), опечатки ("Техничесский").

**3) Рекомендации по улучшению**

1.  **Добавить раздел "Опыт работы" (ВЫСШИЙ ПРИОРИТЕТ):**
    *   Для каждой позиции укажите: период работы, компанию, должность.
    *   Используйте структуру "Достижения/Обязанности" с применением метода STAR (Ситуация - Задача - Действие - Результат).
    *   **Пример для аналитика:** "Автоматизировал еженедельный отчет по Retention с помощью Python и SQL, что сократило время на его подготовку с 6 часов до 30 минут". "Провел A/B-тест изменения onboarding, что повысило конверсию на 7%". Увязывайте действия с бизнес-метриками (LTV, выручка, экономия времени).
    *   Даже если предыдущий опыт не был напрямую аналитическим, опишите проекты или задачи, где вы работали с данными, оптимизировали процессы или анализировали информацию.

2.  **Полностью переработать раздел "Образование":**
    *   Структурируйте: **Высшее/Среднее специальное** и **Курсы/Повышение квалификации**.
    *   Для диплома ГПТУ укажите специальность и год окончания.
    *   Курсы (ATI, Zerokoder) вынесите в отдельный блок "Повышение квалификации" с указанием года, названия курса и организации. Это исправит путаницу с "средним специальным".

3.  **Создать краткий профессиональный профиль (Summary):**
    *   В начале резюме добавьте 3-4 строки о себе. Пример: "Аналитик данных с опытом в продуктовой и маркетинговой аналитике. Владею стеком Python (pandas, NumPy), SQL, ClickHouse для извлечения и обработки данных. Специализируюсь на анализе пользовательского поведения, расчете LTV/ARPU, проведении A/B-тестов. Имею портфолио реализованных проектов."

4.  **Упорядочить и очистить раздел "Навыки":**
    *   Разделите на категории: **Инструменты и Языки** (Python, SQL, ClickHouse, Pandas, Jupyter, Git), **Области знаний** (A/B-тестирование, Мат. статистика, Юнит-экономика, Визуализация данных), **Метрики** (LTV, Retention, ARPU), **Гибкие навыки** (Аналитическое мышление).

**4) Конкретные правки текста резюме**

*   **Название/Заголовок:** `"Аналитик данных (Product/Marketing Analytics)"`
*   **Поле "Salary":** Исправить на числовое значение или формат `"от 150 000 ₽ на руки"`.
*   **Добавить блок "Professional Summary":**
    ```json
    "summary": "Аналитик данных с фокусом на продуктовые и маркетинговые метрики. Опыт автоматизации отчетности, проведения A/B-тестов и анализа пользовательской воронки с помощью Python, SQL и ClickHouse. Нацелен на превращение данных в конкретные бизнес-рекомендации для роста ключевых показателей."
    ```
*   **Переработать блок "education":**
    ```json
    "education": {
      "main": [
        {
          "year": "2014",
          "institution": "ГПТУ-10",
          "degree": "Технический специалист (среднее специальное)"
        }
      ],
      "courses": [
        {
          "year": "2025",
          "institution": "Образовательная платформа ATI",
          "name": "Аналитик данных"
        },
        {
          "year": "2025",
          "institution": "Zerokoder",
          "name": "Автоматизация программирования"
        }
      ]
    }
    ```
*   **Структурировать навыки в новом поле "skills_structured":**
    ```json
    "skills_structured": {
      "tools": ["Python (pandas, NumPy)", "SQL", "ClickHouse", "Jupyter Notebook", "Git/GitHub", "MS Excel"],
      "knowledge": ["A/B и A/A тестирование", "Математическая статистика", "Визуализация данных", "Основы баз данных"],
      "metrics": ["Retention", "LTV", "ARPU", "ARPPU", "Финансовые показатели"],
      "soft_skills": ["Аналитическое мышление", "Работа с большим объемом информации"]
    }
    ```
*   **Главное: Добавить блок "experience"** с детальным описанием последних 2-3 мест работы, следуя рекомендациям из пункта 3.1.

**Итог:** Кандидат обладает сильным техническим потенциалом и актуальными знаниями, но резюме в текущем виде **не продает его опыт**. После добавления детального описания рабочих мест и структурирования информации шансы на приглашение на собеседование возрастут многократно. Портфолио на GitHub является сильным активом — его стоит кратко анонсировать в разделе "Контакты" или "Summary".
